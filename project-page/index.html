TL;DR

The problem

Agent observability platforms capture execution traces but lack meaningful evaluation. Developers must manually inspect large numbers of traces to identify systemic failures. Research-driven error taxonomies are static, rigid, and require extensive human annotation.

What we do

Agentic CLEAR is an open-source Python package that automatically evaluates agent traces at three levels of granularity (system, node, trace), dynamically surfaces recurring failure patterns without predefined taxonomies, and presents them in an interactive dashboard. Just pip install clear-eval and run a single CLI command.

Our Contributions

1

Dynamic Evaluation Methodology

A multi-level method that produces automatic, dynamic, and granular evaluation insights — adapting to the target system and domain without predefined error categories.

A pip-installable Python package (clear-eval) with easy integration, OpenTelemetry-compatible trace ingestion, and an interactive visual dashboard.

Demonstrated efficacy across 4 benchmarks, 7 agentic settings, and tens of thousands of LLM calls — showing alignment with human-annotated errors and predictive signal for task success.

Getting Started

Agentic CLEAR is available as a PyPI package. Install and run the full pipeline with just a few commands:

# Install
pip install clear-eval

# Run evaluation on your agent traces
run-clear-agentic-eval \
    --data-dir data/my_traces \
    --results-dir results \
    --from-raw-traces true \
    --agent-framework langgraph \
    --observability-framework mlflow \
    --eval-model-name gpt-4o \
    --provider openai

# Launch the interactive dashboard
run-clear-agentic-dashboard
# Upload the generated unified_ui_results.zip

CLEAR: LLM Error Analysis

At its core, CLEAR transforms instance-level LLM judge feedback into structured, system-level insights. Rather than just scores, it automatically discovers recurring issues across your dataset, quantifies their frequency, and lets you drill down to concrete examples.

1

Generate Responses

Run your target system on the dataset, or provide pre-existing model outputs.

2

LLM Judge Evaluation

An LLM judge scores each response and provides textual critique explaining what went wrong.

3

Key Point Analysis

Cluster critiques into concise recurring issues via LLM-based KPA, with frequency quantification.

4

Interactive Dashboard

Explore issues, filter by type or score, compare subsets, and drill down to specific examples.

Evaluation Modes

  • General: Issues are discovered dynamically using a general-purpose evaluation prompt — broad, exploratory assessment without prior knowledge.
  • Task-specific: Users provide specific issues as evaluation criteria, guiding the judge while allowing additional discoveries.
  • Static: A predefined list of issues is mapped directly to evaluation texts, without dynamic discovery.

Agentic CLEAR: Multi-Level Agent Evaluation

Extending CLEAR to multi-agent systems, Agentic CLEAR operates above the observability layer, processing execution traces through a two-stage pipeline to produce insights at system, node, and trace levels:

1

Ingest Traces

Accept OpenTelemetry-compatible traces from MLflow, Langfuse, or custom CSV. Convert to unified intermediate representation.

2

Multi-Level Evaluation

An LLM judge evaluates each trace at three levels: step-wise (per LLM call), trace-wise (full trajectory), and rubric-based (auto-generated criteria).

3

CLEAR Aggregation

Cluster and summarize instance-level feedback into system-wide patterns and node-specific recurring issues.

4

Interactive Dashboard

Explore results across System, Node, and Trace views with filtering, drill-down, and path analysis.

The evaluation adapts dynamically to any agent architecture and task domain. Issues are discovered without predefined taxonomies — the system identifies what matters for your specific agent and dataset.

Interactive Dashboard

Dashboard Features

Per-agent CLEAR issues, score distributions, and drill-down into individual records

Common path patterns, success vs. failure analysis, dead-end detection

Temporal Analysis

Agent position analysis and score progression across trajectory steps

Score Prediction

ROC curve analysis for predicting trajectory success from step scores

Results

Discovered Issues

Agentic CLEAR surfaced 195 unique recurring issues across all configurations, revealing both universal error patterns and domain-specific failures — without any benchmark-specific prompting.

Universal Patterns

  • Redundant and inefficient tool usage
  • Insufficient error handling and recovery
  • Incomplete workflows and task abandonment
  • Output formatting and schema compliance failures

Domain-Specific (examples)

  • GAIA: Sourcing and verification failures
  • AppWorld: Incomplete executions, contaminated carts
  • SWE-bench: Monkey-patching, broken diff output
  • TAU-bench: Policy violations, unauthorized actions

Abstract

Agentic systems are becoming more capable: agents define strategies, take actions, and interact with different environments. This autonomy poses serious challenges for overseeing and assessing agent behavior. Most current tools are limited, focusing on observability with basic evaluation or creating a static taxonomy of agent errors.

To address this gap, we present Agentic CLEAR, an automatic, dynamic, and easy-to-use evaluation framework. It produces textual insights into the agent behavior on three levels of granularity: system, trace, and node. Agentic CLEAR operates above the observability layer, enabling seamless integration and featuring an intuitive UI that makes agent evaluation highly accessible.

In our experiments on four benchmarks, seven agentic settings, and tens of thousands of LLM calls, we show that Agentic CLEAR produces high-quality, data-driven, insightful feedback. Our analysis shows strong alignment with human-annotated errors and the ability to predict task success rate.

Output Structure

The pipeline produces a structured output directory with all evaluation results:

results/
└── <run_name>/
    ├── traces_data/ # Preprocessed CSV files
    ├── step_by_step/ # Step-by-step CLEAR results
    │   ├── clear_data/ # Grouped by agent node
    │   └── clear_results/ # Per-agent analysis + issues
    ├── full_trajectory/ # Full trajectory results
    │   ├── task_success/ # Binary success evaluation
    │   ├── full_trajectory/ # Multi-dimensional scores
    │   ├── rubric_generation/ # Auto-generated rubrics
    │   ├── rubric/ # Rubric-based evaluation
    │   └── clear_analysis/ # Root cause + issues
    ├── unified_ui_results.zip # Dashboard input
    └── pipeline_summary.json # Execution metadata
